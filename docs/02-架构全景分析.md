# 第2章：架构全景分析

## 2.1 四层架构总览（框架源码）

```
┌─────────────────────────────────────────────────────────────────┐
│                    04-Application（应用层）                        │
│   CrudAppService + Controllers + Validators + EventHandlers      │
│   + Mapping(Profiles) + Services                                 │
│   引用: 01-Domain, 02-Shared, 03-Infrastructure                  │
│   职责: 编排业务流程，零代码CRUD，DTO映射，控制器基类               │
├─────────────────────────────────────────────────────────────────┤
│                    03-Infrastructure（基础设施层）                  │
│   DbContext + Repositories + Interceptors + Services             │
│   + Middleware(8个) + Configuration(功能拆分扩展)                  │
│   引用: 01-Domain, 02-Shared                                     │
│   职责: 数据持久化，中间件，JWT/Swagger/OpenTelemetry/Serilog配置  │
├─────────────────────────────────────────────────────────────────┤
│                    01-Domain（领域层）                             │
│   Entities + Events + Interfaces + Services                      │
│   引用: 02-Shared                                                │
│   职责: 核心业务逻辑，充血模型实体，领域事件，仓储接口定义            │
├─────────────────────────────────────────────────────────────────┤
│                    02-Shared（共享层）                             │
│   Enums + Models + Exceptions + Attributes + Helpers             │
│   引用: 无（最底层）                                              │
│   职责: 跨层共享的类型定义，ApiResult/PagedResult/DI特性           │
└─────────────────────────────────────────────────────────────────┘
```

> **说明**：框架是纯库项目（4个NuGet包），不包含Web项目。用户创建项目时使用模板自动生成六层架构（含Presentation层）。

## 2.2 依赖关系图

```
04-Application ──────► 03-Infrastructure ──────► 01-Domain ──────► 02-Shared
       │                       │                       │
       └──► 01-Domain ─────────┘                       │
       └──► 02-Shared ──────────────────────────────────┘
```

**关键原则**：
- Shared 层零依赖（最底层）
- Domain 只依赖 Shared
- Infrastructure 依赖 Domain 和 Shared
- Application 依赖 Infrastructure、Domain、Shared（框架内部引用）

## 2.3 实体继承体系

```
BaseEntity<TKey>                          ← 最基类
  │  [Id] 雪花ID自动生成
  │  [DomainEvents] 领域事件收集
  │
  └── BaseAuditEntity<TKey>              ← 审计层
        │  [IsDeleted] 软删除标记
        │  [CreatedTime] 创建时间
        │  [CreatedBy] 创建人
        │  [LastModifiedTime] 修改时间
        │  [LastModifiedBy] 修改人
        │
        └── BaseFullEntity<TKey>          ← 业务层（最常用）
              │  [TenantId] 租户ID
              │  [RowVersion] 乐观锁
              │  [Remark] 备注
              │
              ├── BaseTreeEntity<TEntity, TKey>  ← 树形层
              │     [Name] 节点名称
              │     [ParentId] 父节点ID（可空）
              │     [Path] 路径
              │     [Level] 层级
              │     [Sort] 排序
              │     [IsLeaf] 是否叶子节点
              │     [Move()] 移动节点
              │     [Rename()] 重命名
              │     [SetSort()] 设置排序
              │
              └── BaseComplianceEntity<TKey>     ← 合规层
                    [IsArchived] 是否归档
                    [ArchivedTime] 归档时间
                    [ArchivedBy] 归档人
                    [OperationIp] 操作IP
                    [OperationDevice] 操作设备
                    [DataTraceCode] 数据追溯码
                    [ComplianceArchiveTime] 合规归档时间
```

> **TKey 约束**：所有 `TKey` 必须为 `struct`（值类型），支持 `long`/`int`/`Guid`。
> 默认简写：`BaseEntity` = `BaseEntity<long>`，`BaseFullEntity` = `BaseFullEntity<long>`。

## 2.4 请求处理全流程

```
HTTP Request
  │
  ▼
[Middleware Pipeline]
  ├── Global Exception Middleware      ← 最外层，捕获所有异常
  ├── Serilog Request Logging
  ├── Rate Limit Middleware
  ├── JWT Authentication
  ├── Permission Authorization
  └── ...
  │
  ▼
[Controller]
  │ 接收请求，参数绑定
  │
  ▼
[Application Service]
  │ 编排业务流程
  │ 调用 Repository
  │ 调用 Domain Service
  │ 发布领域事件
  │
  ▼
[Repository]
  │ 数据访问
  │ 调用 UnitOfWork
  │
  ▼
[AppDbContext.SaveChangesAsync()]
  │
  ├── [Interceptor Pipeline]
  │     ├── AuditInterceptor         ← 自动填充创建人/时间
  │     ├── SoftDeleteInterceptor    ← Delete → 设置 IsDeleted
  │     ├── TenantInterceptor        ← 过滤 TenantId
  │     ├── ComplianceInterceptor    ← 合规数据自动归档
  │     ├── DataPermissionInterceptor ← 行级数据权限
  │     ├── SqlLogInterceptor        ← SQL 日志记录
  │     ├── NPlusOneInterceptor      ← N+1 查询检测
  │     ├── OptimisticLockInterceptor ← 乐观锁冲突处理
  │     ├── FieldPermissionInterceptor ← 字段级权限
  │     ├── AuditTrailInterceptor    ← 变更审计追踪
  │     └── EntityConfigurationScanInterceptor ← 自动扫描配置
  │
  ├── [Database Save]
  │
  └── [DispatchDomainEvents]
        └── IDomainEventBus.PublishRangeAsync()
              └── MediatR → EventHandler[]
                    ├── 清除缓存
                    ├── 发送通知
                    ├── 记录日志
                    └── 触发后台任务
  │
  ▼
HTTP Response (ApiResult)
```

## 2.6 拦截器详解

| 拦截器 | 类型 | 触发时机 | 作用 |
|--------|------|---------|------|
| `AuditInterceptor` | `ISaveChangesInterceptor` | SaveChanges | 自动填充 CreatedTime、CreatedBy、LastModifiedTime、LastModifiedBy |
| `SoftDeleteInterceptor` | `ISaveChangesInterceptor` | SaveChanges | 将 Delete 操作转换为设置 IsDeleted=true |
| `TenantInterceptor` | `ISaveChangesInterceptor` | SaveChanges | 插入时自动设置 TenantId；查询时自动过滤当前租户 |
| `ComplianceInterceptor` | `ISaveChangesInterceptor` | SaveChanges | 合规实体的归档时间、操作IP、操作设备自动填充 |
| `DataPermissionInterceptor` | `ISaveChangesInterceptor` | SaveChanges | 行级数据权限过滤 |
| `SqlLogInterceptor` | `IDbCommandInterceptor` | 执行SQL | 记录 SQL 语句和执行时间 |
| `NPlusOneInterceptor` | `IDbCommandInterceptor` | 执行SQL | 检测 N+1 查询问题并警告 |
| `OptimisticLockInterceptor` | `ISaveChangesInterceptor` | SaveChanges | 处理乐观锁并发冲突 |
| `FieldPermissionInterceptor` | `ISaveChangesInterceptor` | SaveChanges | 字段级权限控制 |
| `AuditTrailInterceptor` | `ISaveChangesInterceptor` | SaveChanges | 记录实体变更前后的值 |
| `EntityConfigurationScanInterceptor` | 静态工具类 | OnModelCreating | 自动扫描实体配置并应用 |

## 2.7 内置日志实体

| 实体 | 基类 | 作用 |
|------|------|------|
| `OpLog` | `BaseEntity` | 操作日志（模块、动作、详情、操作人、耗时） |
| `LoginLog` | `BaseEntity` | 登录日志（用户名、IP、UserAgent、登录结果） |

> **注意**：框架不内置用户/角色/权限/租户实体，你需要根据业务需求自行定义。JWT 认证由 `IJwtService` 提供。

## 2.8 服务注册入口

```csharp
var builder = WebApplication.CreateBuilder(args);

builder.AddEfCoreSerilog();

// 一键注册（Swagger + JWT + OpenTelemetry + Cors + 基础设施 + 应用层 全部注册）
builder.Services.AddEfCoreEnterprise(builder.Configuration);

// 覆盖 AppDbContext 注册（使用项目自定义的子类，传入所有拦截器）
builder.Services.AddScoped<AppDbContext>(sp =>
{
    var options = sp.GetRequiredService<DbContextOptions<AppDbContext>>();
    return new YourProject.Infrastructure.Data.AppDbContext(
        options,
        sp.GetRequiredService<AuditInterceptor>(),
        sp.GetRequiredService<SoftDeleteInterceptor>(),
        sp.GetRequiredService<TenantInterceptor>(),
        sp.GetRequiredService<ComplianceInterceptor>(),
        sp.GetRequiredService<IDomainEventBus>());
});

// 自动注入（[Injectable] + FluentValidation + MediatR + AutoMapper）
builder.Services.AddEfCoreAutoInject(assemblies);

var app = builder.Build();

// 一键配置中间件管道（Swagger/Cors/Serilog/中间件/认证/授权）
app.UseEfCorePipeline(app.Environment.IsDevelopment());

app.MapHealthChecks("/health");
app.Run();
```

### 功能拆分扩展

每个功能都有独立的扩展方法，可按需自由组合：

| 扩展方法 | 文件 | 功能 |
|---------|------|------|
| `AddEfCoreSerilog()` | `SerilogExtensions.cs` | Serilog 结构化日志 |
| `AddEfCoreSwagger()` | `SwaggerExtensions.cs` | Swagger API 文档 |
| `AddEfCoreJwt()` | `JwtExtensions.cs` | JWT 认证 |
| `AddEfCoreOpenTelemetry()` | `OpenTelemetryExtensions.cs` | OpenTelemetry 可观测性 |
| `AddEfCoreCors()` | `CorsExtensions.cs` | 跨域配置 |
| `AddEfCoreControllers()` | `ControllerExtensions.cs` | Controller + HttpClient |
| `AddDevMode()` | `DevModeExtensions.cs` | 开发模式 |
| `AddInfrastructureServices()` | `InfrastructureServiceExtensions.cs` | 基础设施服务 |
| `AddApplicationServices()` | `ApplicationServiceExtensions.cs` | 应用层服务 |
| `AddEfCoreAutoInject()` | `EfCoreEnterpriseExtensions.cs` | [Injectable]自动注册 |
| `UseEfCoreMiddleware()` | `MiddlewareExtensions.cs` | 六合一中间件管道 |
| `UseEfCoreGrayRelease()` | `MiddlewareExtensions.cs` | 灰度发布中间件 |
| `UseEfCoreValidation()` | `MiddlewareExtensions.cs` | 参数验证过滤器 |
| `UseDevMode()` | `DevModeExtensions.cs` | 开发模式启用 |
| `UseEfCorePipeline()` | `EfCoreEnterpriseExtensions.cs` | 管道总入口 |

```csharp
// 分步注册（完全自定义，跳过框架默认配置）
builder.Services.AddEfCoreControllers();
builder.Services.AddEfCoreSwagger(configuration);
builder.Services.AddEfCoreJwt(configuration);
builder.Services.AddEfCoreOpenTelemetry(configuration);
builder.Services.AddEfCoreCors();
builder.Services.AddInfrastructureServices(connectionString,
    enableRedis: true,
    redisConnection: "localhost:6379",
    enableHangfire: true);
```

## 2.10 项目模板 vs 框架源码

| 层 | 框架提供 | 模板提供 |
|----|---------|---------|
| 01-Domain | 实体基类、仓储接口、领域事件基类、日志实体 | 自定义实体（Product/Category/News） |
| 02-Shared | ApiResult、PagedResult、异常、枚举、DI特性 | 无（全部使用框架） |
| 03-Contracts | 无 | 自定义 DTO |
| 04-Application | `CrudAppService`、`BaseApiController`、`CrudController`、`BaseProfile`、MediatR配置 | 自定义 Service、Profile、Validator、EventHandler |
| 05-Infrastructure | `AppDbContext`、`SuperRepository`、`TreeRepository`、11个拦截器、17+服务、**8个中间件**、**功能拆分扩展**（Swagger/JWT/OpenTelemetry/Serilog/Cors） | 自定义 DbContext（继承框架）、实体配置、种子数据 |
| 06-Presentation | 无（框架不包含Web项目） | Program.cs（使用 `AddEfCoreEnterprise`/`UseEfCorePipeline`）、Controller、配置文件 |

> **重要变更**：原框架的 Presentation 层已删除，其中间件已迁移到 Infrastructure 层，控制器基类已迁移到 Application 层。用户在模板的 Program.cs 中只需调用框架提供的扩展方法即可。
