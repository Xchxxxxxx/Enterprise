## 8.1 注入缓存服务

```csharp
[Injectable]
public class ProductService : BaseService
{
    private readonly ICacheService _cache;

    public ProductService(ICacheService cache)
    {
        _cache = cache;
    }
}
```

## 8.2 基础用法

```csharp
// ===== 设置缓存（带过期时间） =====
await _cache.SetAsync("user:1", userDto, TimeSpan.FromMinutes(30));

// ===== 读取缓存 =====
var user = await _cache.GetAsync<UserDto>("user:1");

// ===== 读取（不存在则写入，推荐） =====
var user = await _cache.GetOrSetAsync("user:1",
    async () => {
        var entity = await _userRepo.GetByIdAsync(1);
        return entity is null ? null : _mapper.Map<UserDto>(entity);
    },
    TimeSpan.FromMinutes(30));

// ===== 删除 =====
await _cache.RemoveAsync("user:1");

// ===== 按前缀批量删除 =====
await _cache.RemoveByPrefixAsync("user:");

// ===== 判断是否存在 =====
var exists = await _cache.ExistsAsync("user:1");
```

## 8.3 缓存穿透防护（BloomFilter）

**框架自动启用 BloomFilter**，无需额外配置。

**原理**：BloomFilter 是一个概率型数据结构，可以快速判断 key 是否存在。

**效果**：查询不存在的 key 时，BloomFilter 直接返回 false，**不会穿透到数据库**。

```csharp
// 正常查缓存，BloomFilter 自动保护
var user = await _cache.GetAsync<UserDto>("user:999999");
// → 返回 null，但不会查数据库，BloomFilter 直接拦截
```

**适用场景**：防止恶意查询不存在的 ID 击穿缓存。

## 8.4 分布式锁

```csharp
// 方式一：using 自动释放
await using (await _cache.LockAsync("order:lock:123", TimeSpan.FromSeconds(10)))
{
    // 你的业务逻辑
    // 锁自动在 using 结束时释放
}

// 方式二：手动控制
var locked = await _cache.TryLockAsync("order:lock:123", TimeSpan.FromSeconds(10));
if (locked)
{
    try
    {
        // 业务逻辑
    }
    finally
    {
        await _cache.UnlockAsync("order:lock:123");
    }
}
```

**典型场景**：防止并发操作库存

```csharp
public async Task DeductStockAsync(long productId, int quantity)
{
    var lockKey = $"stock:lock:{productId}";
    await using (await _cache.LockAsync(lockKey, TimeSpan.FromSeconds(10)))
    {
        var product = await _productRepo.GetByIdAsync(productId)
            ?? throw new NotFoundException("产品不存在");

        if (product.Stock < quantity)
            throw new BusinessException("库存不足");

        product.ChangeStock(product.Stock - quantity);
        await _productRepo.UpdateAsync(product);
        await _unitOfWork.SaveChangesAsync();
    }
}
```

## 8.6 多级缓存架构

框架自动实现 L1（内存）→ L2（Redis）多级缓存：

```
请求 → L1 内存缓存（极快）
  ↓ 未命中
L2 Redis 缓存（快）
  ↓ 未命中
BloomFilter 检查
  ↓ 可能存在
数据库查询
  ↓
写入 L2 → 写入 L1
```

**配置 Redis**：

```csharp
// 在 Program.cs 中启用 Redis
builder.Services.AddInfrastructureServices(
    connectionString,
    enableRedis: true,
    redisConnection: "localhost:6379");
```

如果未启用 Redis，框架自动使用内存缓存作为降级方案。

## 8.8 缓存最佳实践

| 实践 | 说明 |
|------|------|
| 使用 `GetOrSetAsync` | 避免缓存击穿，一步完成"读或写" |
| 设置合理的过期时间 | 不常变的数据可以设置更长（如 30 分钟） |
| 使用前缀命名 | `user:1`、`product:1`、`order:list:1` |
| 更新时清除缓存 | 在事件处理器中调用 `RemoveAsync` |
| 分布式锁保护 | 防止缓存击穿到数据库 |
| 缓存预热 | 应用启动时加载热点数据 |
