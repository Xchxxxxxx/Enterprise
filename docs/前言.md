# 前言：EfCore.Enterprise 是什么

EfCore.Enterprise 是一个**企业级超高性能 EF Core 开发框架**，基于 .NET 8，采用 DDD（领域驱动设计）六层架构，封装了 EF Core 在企业开发中几乎所有的重复性工作。

## 一句话概括

**你只写实体和业务逻辑，框架帮你搞定一切基础设施。**

## 核心设计理念

| 理念 | 解释 |
|------|------|
| **充血模型** | 实体不仅有属性，还有行为方法（`Order.Pay()` 而不是 `order.Status = Paid`） |
| **约定优于配置** | 继承 `BaseFullEntity` 自动获得审计、软删除、租户隔离、乐观锁 |
| **零代码 CRUD** | 继承 `CrudAppService` 即可获得完整的分页、增删改查 |
| **自动 DI** | 加上 `[Injectable]` 自动注册，无需手动写 `AddScoped` |
| **拦截器驱动** | 11 个 EF Core 拦截器自动处理审计、软删除、租户、合规等 |
| **领域事件** | `SaveChanges` 后自动分发事件，通过 MediatR 异步处理 |

## 框架包含什么

### NuGet 包结构

```
EfCore.Enterprise（总包，一行安装）
├── EfCore.Enterprise.Domain         领域层
├── EfCore.Enterprise.Shared         共享层
├── EfCore.Enterprise.Infrastructure 基础设施层
└── EfCore.Enterprise.Application    应用层
```

### 领域层

- **5 种实体基类**：`BaseEntity` / `BaseAuditEntity` / `BaseFullEntity` / `BaseTreeEntity` / `BaseComplianceEntity`
- **3 种仓储接口**：`IRepository` / `ISuperRepository` / `ITreeRepository`
- **领域事件总线**：`IDomainEventBus` + `DomainEvent`
- **领域服务接口**：`IDomainService` 标记接口

### 共享层

- **统一响应**：`ApiResult` / `ApiResult<T>` / `PagedResult<T>`
- **分页请求**：`PagedRequest` / `PagedRequest<TFilter>`
- **异常体系**：`AppException`、`BusinessException`、`NotFoundException`、`ValidationException` 等
- **自动 DI**：`[Injectable]` 特性 + `[IgnoreInjectable]` 排除
- **模块化注册**：`IModule`

### 基础设施层

- **AppDbContext**：自动配置全局过滤器、领域事件分发
- **11 个拦截器**：审计、软删除、租户、合规、数据权限、SQL日志、N+1检测、乐观锁、字段权限、审计追踪、实体配置
- **SuperRepository**：完整 CRUD + 分页 + 游标分页 + 批量操作 + 软删除 + 锁定 + 归档
- **TreeRepository**：树形结构查询、移动、层级管理
- **8 个中间件**：全局异常、请求日志、限流、幂等、性能监控、操作日志、灰度发布、参数验证
- **17+ 内置服务**：缓存、分布式锁、雪花ID、后台任务、限流、熔断、性能监控、告警、代码生成、插件、幂等、死信队列、合规审计、BloomFilter、对象池、热数据缓存、开发模式
- **模块化配置扩展**：`AddEfCoreSwagger()`、`AddEfCoreJwt()`、`AddEfCoreOpenTelemetry()` 等

### 应用层

- **CrudAppService**：泛型 CRUD 服务，零代码完成增删改查
- **BaseService**：服务基类，提供 UnitOfWork、Mapper
- **BaseProfile**：AutoMapper 配置基类，只需实现 `Configure()` 方法
- **JWT 服务**：Token 生成、验证、刷新

## 谁适合用

- 中小团队快速启动新项目
- 需要标准化开发流程的企业
- 想从 ABP 迁移但嫌 ABP 太重的团队
- 对性能有极致要求的场景（游标分页、BloomFilter、对象池、查询预编译缓存）

4. **第4章 仓储与查询** → 学会数据访问
