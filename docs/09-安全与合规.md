# 第9章：安全与合规

## 9.1 权限控制

在 Controller 或 Action 上加 `[Permission]` 特性：

```csharp
[Permission("System.User.Query")]
public async Task<PagedResult<UserDto>> GetPage(PagedRequest request)
{
    return await _userService.GetPageAsync(request);
}

[Permission("System.User.Create")]
public async Task<ApiResult<UserDto>> Create(CreateUserRequest request)
{
    return ApiResult<UserDto>.Ok(await _userService.CreateAsync(request));
}

[Permission("System.User.Delete")]
public async Task<ApiResult> Delete(long id)
{
    await _userService.DeleteAsync(id);
    return ApiResult.Ok();
}
```

**权限编码规则：** `模块.实体.操作`，如 `System.User.Create`、`Order.Order.Cancel`。

## 9.2 数据权限（行级隔离）

```csharp
// 获取当前用户可见的角色 ID
[Injectable]
public class UserService : BaseService
{
    private readonly IDataPermissionProvider _dataPermission;

    public UserService(IDataPermissionProvider dataPermission)
    {
        _dataPermission = dataPermission;
    }

    public async Task<List<UserDto>> GetVisibleUsers()
    {
        // 获取当前用户能看到的角色范围
        var roleIds = await _dataPermission.GetCurrentRoleIds();

        // 只查询有权限的数据
        var users = await _userRepo.GetListAsync(u =>
            u.UserRoles.Any(ur => roleIds.Contains(ur.RoleId)));

        return _mapper.Map<List<UserDto>>(users);
    }
}
```

`IDataPermissionProvider` 接口：

```csharp
public interface IDataPermissionProvider
{
    long? GetCurrentUserId();
    IEnumerable<long> GetCurrentRoleIds();
    IEnumerable<string> GetCurrentPermissions();
}
```

框架的 `DataPermissionInterceptor` 自动在数据库查询时应用行级过滤。

## 9.4 审计日志

**框架自动记录**，不需要你写代码：

| 记录内容 | 触发条件 | 拦截器 |
|---------|---------|--------|
| 谁创建了数据 | 新增实体 | `AuditInterceptor` |
| 谁修改了数据 | 更新实体 | `AuditInterceptor` |
| 修改了哪些字段 | 更新实体 | `AuditTrailInterceptor` |
| 操作 IP | 所有请求 | `AuditInterceptor` |
| 谁删除了数据 | 删除实体 | `SoftDeleteInterceptor` |
| 数据追溯码 | 合规实体 | `ComplianceInterceptor` |

## 9.5 参数验证

**自动校验**，只需写 DTO Validator：

```csharp
public class CreateUserRequestValidator : AbstractValidator<CreateUserRequest>
{
    public CreateUserRequestValidator()
    {
        RuleFor(x => x.Username)
            .NotEmpty().WithMessage("用户名不能为空")
            .MinimumLength(3).MaximumLength(50);

        RuleFor(x => x.Password)
            .NotEmpty().MinimumLength(6)
            .Matches("[A-Z]").WithMessage("密码必须包含大写字母")
            .Matches("[a-z]").WithMessage("密码必须包含小写字母")
            .Matches("[0-9]").WithMessage("密码必须包含数字");

        RuleFor(x => x.Email)
            .EmailAddress().When(x => !string.IsNullOrEmpty(x.Email));
    }
}
```

**框架自动在请求进入 Controller 前校验**，校验失败返回 400 + 详细错误信息。

## 9.6 全局异常处理

**框架自动捕获所有异常**，统一返回格式：

```json
{
  "success": false,
  "code": "ORDER_NOT_FOUND",
  "message": "订单不存在",
  "traceId": "abc123"
}
```

你只需要在业务代码中抛出对应异常：

```csharp
throw new NotFoundException("订单不存在");
throw new BusinessException("订单金额不能为负数");
throw new ConcurrentConflictException("数据已被修改");
```

## 9.7 合规追溯

继承 `BaseComplianceEntity` 的实体自动获得：

| 字段 | 说明 |
|------|------|
| `DataTraceCode` | 数据追溯码，唯一标识一条数据 |
| `OperationIp` | 操作者 IP 地址 |
| `OperationDevice` | 操作者设备信息 |
| `IsArchived` | 是否已归档 |
| `ArchivedTime` | 归档时间 |
| `ArchivedBy` | 归档人 |
| `ComplianceArchiveTime` | 合规归档时间 |

用于金融、医疗等需要完整数据追溯的合规场景。

## 9.8 JWT 认证

```csharp
// 框架内置 IJwtService
public interface IJwtService
{
    TokenValidationParameters GetValidationParameters();
    string GenerateAccessToken(long userId, string username, IEnumerable<string> roles, IEnumerable<string> permissions);
    string GenerateRefreshToken();
    ClaimsPrincipal? ValidateToken(string token);
    string? GetJwtId(string token);
}
```

配置 JWT：

```json
{
  "Jwt": {
    "SecretKey": "至少32个字符的密钥",
    "Issuer": "EfCore.Enterprise",
    "Audience": "EfCore.Enterprise",
    "ExpireMinutes": 1440
  }
}
```

## 9.9 内置日志实体

框架内置了操作日志和登录日志实体：

| 实体 | 基类 | 说明 |
|------|------|------|
| `OpLog` | `BaseEntity` | 操作日志（模块、动作、详情、操作人、耗时） |
| `LoginLog` | `BaseEntity` | 登录日志（用户名、IP、UserAgent、登录结果） |

> **注意**：框架不内置用户/角色/权限/租户实体，你需要根据业务需求自行定义。JWT 认证由 `IJwtService` 提供。
