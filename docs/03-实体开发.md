# 第3章：实体开发

## 3.1 实体基类继承体系

```
BaseEntity<TKey>
  └── BaseAuditEntity<TKey>        (+审计)
        └── BaseFullEntity<TKey>   (+多租户+乐观锁+备注)
              ├── BaseTreeEntity<TEntity, TKey>  (+树形结构)
              └── BaseComplianceEntity<TKey>     (+合规归档)
```

## 3.2 选择基类指南

| 基类 | 适用场景 | 示例 |
|------|---------|------|
| `BaseEntity` | 简单配置表、字典表、关联表 | 字典项、用户-角色关联 |
| `BaseAuditEntity` | 需要审计追踪的表 | 操作日志、登录日志 |
| `BaseFullEntity` | **业务表（最常用）** | 订单、产品、用户 |
| `BaseTreeEntity` | 树形结构 | 组织架构、分类、菜单 |
| `BaseComplianceEntity` | 合规场景 | 金融交易、医疗记录 |

## 3.3 充血模型规范

**核心规则：属性全部 `private set`，只能通过构造函数和方法修改。**

```csharp
using EfCore.Enterprise.Domain.Entities;
using EfCore.Enterprise.Domain.Events;

namespace MyApp.Domain.Entities;

public class Order : BaseFullEntity
{
    // ===== 1. 私有无参构造函数（EF Core 需要，永远为空） =====
    private Order() { }

    // ===== 2. 公开构造函数（创建时调用，初始化必填字段） =====
    public Order(string orderNo, long customerId, decimal amount)
    {
        OrderNo = orderNo;
        CustomerId = customerId;
        Amount = amount;
        Status = OrderStatus.Pending;
        OrderTime = DateTimeOffset.UtcNow;
    }

    // ===== 3. 属性：全部 private set =====
    public string OrderNo { get; private set; } = null!;
    public long CustomerId { get; private set; }
    public decimal Amount { get; private set; }
    public OrderStatus Status { get; private set; }
    public DateTimeOffset OrderTime { get; private set; }
    public DateTimeOffset? PaidTime { get; private set; }
    public DateTimeOffset? ShippedTime { get; private set; }
    public string? CancelReason { get; private set; }

    // ===== 4. 领域方法：封装业务逻辑 =====
    public void Pay()
    {
        if (Status != OrderStatus.Pending)
            throw new InvalidOperationException($"只能对待支付订单付款，当前状态: {Status}");

        Status = OrderStatus.Paid;
        PaidTime = DateTimeOffset.UtcNow;

        // 发布领域事件
        AddDomainEvent(new OrderPaidDomainEvent(Id, OrderNo, Amount));
    }

    public void Ship()
    {
        if (Status != OrderStatus.Paid)
            throw new InvalidOperationException("只能对已支付订单发货");

        Status = OrderStatus.Shipped;
        ShippedTime = DateTimeOffset.UtcNow;

        AddDomainEvent(new OrderShippedDomainEvent(Id, OrderNo));
    }

    public void Cancel(string reason)
    {
        if (Status == OrderStatus.Shipped)
            throw new InvalidOperationException("已发货订单不能取消");

        if (string.IsNullOrWhiteSpace(reason))
            throw new ArgumentException("取消原因不能为空");

        Status = OrderStatus.Cancelled;
        CancelReason = reason;

        AddDomainEvent(new OrderCancelledDomainEvent(Id, OrderNo, reason));
    }

    // 简单属性也可以直接 set（不需要业务规则时）
    public void SetRemark(string? remark) => Remark = remark;
}
```

## 3.4 使用实体

```csharp
// ===== 创建 =====
var order = new Order("ORD-20260703-001", customerId, 99.99m);
await _orderRepo.AddAsync(order);
await _unitOfWork.SaveChangesAsync();

// ===== 付款：调用领域方法，不是直接改属性 =====
order.Pay();
await _orderRepo.UpdateAsync(order);
await _unitOfWork.SaveChangesAsync();

// ===== 取消 =====
order.Cancel("客户要求退款");
await _orderRepo.UpdateAsync(order);
await _unitOfWork.SaveChangesAsync();
```

## 3.5 简单实体（News）

不需要复杂业务逻辑的实体，可以简化：

```csharp
using EfCore.Enterprise.Domain.Entities;

namespace MyApp.Domain.Entities;

public class News : BaseEntity
{
    public string Title { get; set; } = null!;
    public string Content { get; set; } = null!;
    public string Status { get; set; } = "Draft";
    public DateTime? PublishTime { get; set; }
}
```

## 3.6 树形实体（Category）

```csharp
using EfCore.Enterprise.Domain.Entities;

namespace MyApp.Domain.Entities;

public class Category : BaseTreeEntity<Category>
{
    public string Code { get; set; } = null!;
    public string? Description { get; set; }
}
```

`BaseTreeEntity<Category>` 自动提供：
- `Move(newParentId)` — 移动节点
- `Rename(newName)` — 重命名
- `SetSort(sort)` — 设置排序
- `Name`、`ParentId`、`Path`、`Level`、`Sort`、`IsLeaf` 字段

## 3.7 使用其他主键类型

## 3.9 领域事件发布

```csharp
public class Order : BaseFullEntity
{
    public void Pay()
    {
        Status = OrderStatus.Paid;

        // 方式一：直接创建事件对象
        AddDomainEvent(new OrderPaidDomainEvent
        {
            OrderId = Id,
            OrderNo = OrderNo,
            Amount = Amount
        });
    }

    public void Cancel(string reason)
    {
        Status = OrderStatus.Cancelled;

        // 方式二：通过构造函数创建事件
        AddDomainEvent(new OrderCancelledDomainEvent(Id, OrderNo, reason));
    }
}
```

事件在 `SaveChangesAsync()` 成功后自动分发，详见第5章。

## 3.10 实体配置

```csharp
// 05-Infrastructure/Data/Configurations/OrderConfiguration.cs
using Microsoft.EntityFrameworkCore;
using Microsoft.EntityFrameworkCore.Metadata.Builders;

namespace MyApp.Infrastructure.Data.Configurations;

public class OrderConfiguration : IEntityTypeConfiguration<Order>
{
    public void Configure(EntityTypeBuilder<Order> builder)
    {
        builder.ToTable("Orders");

        builder.HasKey(x => x.Id);
        builder.Property(x => x.OrderNo).HasMaxLength(50).IsRequired();
        builder.Property(x => x.Amount).HasPrecision(18, 2);
        builder.Property(x => x.Status).HasConversion<string>().HasMaxLength(20);

        builder.HasIndex(x => x.OrderNo).IsUnique();
        builder.HasIndex(x => x.CustomerId);
        builder.HasIndex(x => x.CreatedTime);
    }
}
```

## 3.12 新增实体完整步骤

1. 在 `01-Domain/Entities/` 创建实体类，继承 `BaseFullEntity`
2. 属性全部 `private set`，写 `private` 无参构造函数
3. 写公开构造函数和领域方法
4. 在 `05-Infrastructure/Data/Configurations/` 创建实体配置
5. 在 `05-Infrastructure/Data/AppDbContext.cs` 添加 `DbSet`
6. 创建迁移并更新数据库
