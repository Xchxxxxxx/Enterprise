# 第5章：领域事件与领域服务

## 5.1 领域事件原理

**领域事件**是实体状态变更时自动触发的消息。框架在 `SaveChangesAsync()` 成功后自动分发事件，你不需要手动调用。

### 执行流程

```
你的代码: order.Pay()
  ↓
order.AddDomainEvent(new OrderPaidDomainEvent(...))
  ↓
SaveChangesAsync() 提交事务
  ↓
AppDbContext 收集所有 ChangeTracker 中的 DomainEvents
  ↓
清除所有实体的 DomainEvents 列表
  ↓
IDomainEventBus.PublishRangeAsync(events)
  ↓
MediatR 路由到对应 INotificationHandler<T> 实现
  ↓
Handler 执行（清除缓存、发通知、记录日志、触发后台任务...）
```

## 5.2 定义领域事件

```csharp
// 01-Domain/Events/OrderPaidDomainEvent.cs
using EfCore.Enterprise.Domain.Events;

namespace MyApp.Domain.Events;

public class OrderPaidDomainEvent : DomainEvent
{
    public long OrderId { get; set; }
    public string OrderNo { get; set; } = null!;
    public decimal Amount { get; set; }

    public OrderPaidDomainEvent() { }

    public OrderPaidDomainEvent(long orderId, string orderNo, decimal amount)
    {
        OrderId = orderId;
        OrderNo = orderNo;
        Amount = amount;
    }
}
```

## 5.3 在实体中发布事件

```csharp
public class Order : BaseFullEntity
{
    public void Pay()
    {
        if (Status != OrderStatus.Pending)
            throw new InvalidOperationException("只能对待支付订单付款");

        Status = OrderStatus.Paid;
        PaidTime = DateTimeOffset.UtcNow;

        // 发布事件
        AddDomainEvent(new OrderPaidDomainEvent(Id, OrderNo, Amount));
    }

    public void Cancel(string reason)
    {
        Status = OrderStatus.Cancelled;
        CancelReason = reason;

        // 发布事件
        AddDomainEvent(new OrderCancelledDomainEvent(Id, OrderNo, reason));
    }
}
```

## 5.4 编写事件处理器

```csharp
// 04-Application/Orders/EventHandlers/OrderPaidEventHandler.cs
using MediatR;
using EfCore.Enterprise.Infrastructure.Caching;
using Microsoft.Extensions.Logging;
using MyApp.Domain.Events;

namespace MyApp.Application.Orders.EventHandlers;

public class OrderPaidEventHandler : INotificationHandler<OrderPaidDomainEvent>
{
    private readonly ICacheService _cache;
    private readonly ILogger<OrderPaidEventHandler> _logger;

    public OrderPaidEventHandler(ICacheService cache, ILogger<OrderPaidEventHandler> logger)
    {
        _cache = cache;
        _logger = logger;
    }

    public async Task Handle(OrderPaidDomainEvent notification, CancellationToken ct)
    {
        _logger.LogInformation(
            "订单已支付: OrderId={OrderId}, OrderNo={OrderNo}, Amount={Amount}",
            notification.OrderId, notification.OrderNo, notification.Amount);

        // 清除相关缓存
        await _cache.RemoveAsync($"order:{notification.OrderId}", ct);
        await _cache.RemoveByPrefixAsync("order:list:", ct);
    }
}
```

## 5.5 一个实体触发多个事件

```csharp
public class Order : BaseFullEntity
{
    public void Ship()
    {
        Status = OrderStatus.Shipped;
```

## 5.6 一个事件多个处理器

```csharp
// 处理器1：清除缓存
public class OrderPaidCacheHandler : INotificationHandler<OrderPaidDomainEvent>
{
    public async Task Handle(OrderPaidDomainEvent e, CancellationToken ct)
    {
        await _cache.RemoveAsync($"order:{e.OrderId}", ct);
    }
}

// 处理器2：发送通知
public class OrderPaidNotificationHandler : INotificationHandler<OrderPaidDomainEvent>
{
    public async Task Handle(OrderPaidDomainEvent e, CancellationToken ct)
    {
        await _notificationService.SendPaymentSuccessAsync(e.OrderNo);
    }
}

// 处理器3：更新统计
public class OrderPaidStatisticsHandler : INotificationHandler<OrderPaidDomainEvent>
{
    public async Task Handle(OrderPaidDomainEvent e, CancellationToken ct)
    {
        await _statisticsService.IncrementOrderCountAsync();
    }
}
```

## 5.7 树形节点移动事件

框架内置 `TreeNodeMovedEvent<TKey>`，移动节点时自动触发：

```csharp
// 在 BaseTreeEntity.Move() 中自动发布
public class TreeNodeMovedEvent<TKey> : DomainEvent where TKey : struct
{
    public TKey NodeId { get; set; }
    public string NodeName { get; set; } = string.Empty;
    public TKey? OldParentId { get; set; }
    public TKey? NewParentId { get; set; }
}

// 处理移动事件
public class CategoryMovedHandler : INotificationHandler<TreeNodeMovedEvent<long>>
{
    public async Task Handle(TreeNodeMovedEvent<long> e, CancellationToken ct)
    {
        _logger.LogInformation(
            "分类移动: {NodeName} ({NodeId}) from {OldParent} to {NewParent}",
            e.NodeName, e.NodeId, e.OldParentId, e.NewParentId);
    }
}
```

## 5.8 领域服务 vs 实体方法

| 场景 | 放在哪里 | 示例 |
|------|---------|------|
| 修改单个实体自己的属性 | 实体方法 | `order.Pay()` |
| 需要查数据库判断 | 领域服务 | `IsPasswordStrongEnough()` |
| 涉及多个实体协作 | 领域服务 | `AssignDefaultRole(user)` |
| 需要调用外部服务 | 领域服务 | `SendSms(phone, code)` |
| 复杂业务规则校验 | 领域服务 | `CanPlaceOrder(customer)` |

## 5.9 编写领域服务

```csharp
// 接口：01-Domain/Interfaces/IOrderDomainService.cs
using MyApp.Domain.Entities;

namespace MyApp.Domain.Interfaces;

public interface IOrderDomainService
{
    Task<bool> CanCancelAsync(long orderId);
    Task CancelOrderAsync(Order order, string reason);
}

// 实现：01-Domain/Services/OrderDomainService.cs
using EfCore.Enterprise.Domain.Interfaces;
using MyApp.Domain.Entities;
using MyApp.Domain.Interfaces;

namespace MyApp.Domain.Services;

public class OrderDomainService : IOrderDomainService
{
    private readonly ISuperRepository<Order> _orderRepo;

    public OrderDomainService(ISuperRepository<Order> orderRepo)
    {
        _orderRepo = orderRepo;
    }

    public async Task<bool> CanCancelAsync(long orderId)
    {
        var order = await _orderRepo.GetByIdAsync(orderId);
        return order != null && order.Status != OrderStatus.Shipped;
    }

    public async Task CancelOrderAsync(Order order, string reason)
    {
        if (order.Status == OrderStatus.Shipped)
            throw new InvalidOperationException("已发货订单不能取消");

        order.Cancel(reason);
        await _orderRepo.UpdateAsync(order);
        await _orderRepo.SaveChangesAsync();
    }
}
```

## 5.11 事件处理注意事项

1. **不要在事件处理器中修改触发事件的实体**（会导致循环依赖）
2. **事件处理器是异步的**，不会阻塞 HTTP 响应
3. **事件处理器中的异常不会影响主流程**（框架已做异常隔离）
4. **事件处理器可以注入任何框架服务**（ICacheService、IBackgroundJobService 等）
