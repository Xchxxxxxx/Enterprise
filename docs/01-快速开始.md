# 第1章：快速开始

## 1.1 环境要求

| 组件 | 版本 |
|------|------|
| .NET SDK | 8.0+ |
| 数据库 | MySQL 8.0+ / SQL Server 2019+ / PostgreSQL 14+ |
| Redis（可选） | 6.0+（缓存和分布式锁需要） |

## 1.2 安装框架

### 方式一：从 GitHub 下载安装（推荐）

框架已发布到 GitHub Releases，模板自带框架包，无需额外配置：

```powershell
# 下载模板包
curl -L -o EfCore.Enterprise.Templates.1.0.1.nupkg --ssl-no-revoke "https://github.com/Xchxxxxxx/Enterprise/releases/download/v1.0.1/EfCore.Enterprise.Templates.1.0.1.nupkg"

# 安装模板
dotnet new install EfCore.Enterprise.Templates.1.0.1.nupkg

# 创建项目
dotnet new ef-enterprise -n MyProject --database mysql
```

> **说明**：模板 `local-packages` 目录已内置框架 NuGet 包，项目的 `nuget.config` 自动指向 `./local-packages`，无论模板装在哪里都能正确引用。

### 方式二：安装 CLI 工具

```powershell
# 下载 CLI 工具包
curl -L -o EfCore.Enterprise.1.0.1.nupkg --ssl-no-revoke "https://github.com/Xchxxxxxx/Enterprise/releases/download/v1.0.1/EfCore.Enterprise.1.0.1.nupkg"

# 安装 CLI 工具
dotnet tool install -g EfCore.Enterprise.Cli --add-source "."

# 验证
ef-cli help
```

### 方式三：本地 nupkgs 目录（开发环境）

在项目根目录执行打包后，直接从 `nupkgs/` 目录引用：

```powershell
# 打包框架
dotnet build EfCore.Enterprise.sln -c Release
dotnet pack EfCore.Enterprise.sln -c Release -o nupkgs
dotnet pack tools\EfCore.Enterprise\EfCore.Enterprise.csproj -c Release -o nupkgs
dotnet pack tools\EfCore.Enterprise.Templates\EfCore.Enterprise.Templates.csproj -c Release -o nupkgs
```

### 方式四：从本地 nupkgs 安装单个包

```bash
dotnet add package EfCore.Enterprise.Domain -s "d:\自建\项目\ef\nupkgs"
dotnet add package EfCore.Enterprise.Shared -s "d:\自建\项目\ef\nupkgs"
dotnet add package EfCore.Enterprise.Infrastructure -s "d:\自建\项目\ef\nupkgs"
dotnet add package EfCore.Enterprise.Application -s "d:\自建\项目\ef\nupkgs"
```

### 方式五：安装项目模板（本地开发）

```bash
dotnet new install "d:\自建\项目\ef\nupkgs\EfCore.Enterprise.Templates.1.0.1.nupkg"
```

## 1.3 创建项目

```bash
# 使用模板创建六层架构项目
dotnet new ef-enterprise -n MyApp

# 指定数据库类型
dotnet new ef-enterprise -n MyApp Database=sqlserver
dotnet new ef-enterprise -n MyApp Database=postgresql
```

## 1.5 配置数据库与功能

编辑 `06-Presentation/appsettings.json`，以下是最小配置：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Port=3306;Database=MyApp_Db;User=root;Password=123456;Charset=utf8mb4;"
  },
  "Jwt": {
    "SecretKey": "your-super-secret-key-at-least-32-characters-long",
    "Issuer": "MyApp",
    "Audience": "MyApp",
    "AccessTokenExpirationMinutes": 30,
    "RefreshTokenExpirationDays": 7
  },
  "Serilog": {
    "MinimumLevel": {
      "Default": "Information",
      "Override": {
        "Microsoft": "Warning",
        "System": "Warning"
      }
    },
    "WriteTo": [
      { "Name": "Console" },
      {
        "Name": "File",
        "Args": {
          "path": "logs/log-.txt",
          "rollingInterval": "Day",
          "retainedFileCountLimit": 30
        }
      }
    ],
    "Enrich": [ "FromLogContext", "WithMachineName", "WithThreadId" ]
  },
  "AllowedHosts": "*"
}
```

> **完整配置说明**请参阅 [附录 § E. 完整配置项参考](附录.md#e-完整配置项参考)，包含 Redis、Snowflake、OpenTelemetry、监控参数等所有可选配置节。

| 配置节 | 必填 | 说明 |
|--------|:----:|------|
| `ConnectionStrings:DefaultConnection` | ✅ | 数据库连接字符串 |
| `Jwt:SecretKey` | ✅ | JWT 密钥，**至少 32 个字符** |
| `Jwt:Issuer` | ✅ | JWT 签发者 |
| `Jwt:Audience` | ✅ | JWT 受众 |
| `Jwt:AccessTokenExpirationMinutes` | ⭕ | AccessToken 过期分钟数（默认 30） |
| `Jwt:RefreshTokenExpirationDays` | ⭕ | RefreshToken 过期天数（默认 7） |
| `Serilog` | ⭕ | 日志配置（控制台 + 文件） |

## 1.6 数据库迁移与启动

### 1. 创建工厂类（一行代码）

在 `05-Infrastructure/Data/` 下新建 `MyAppDbContextFactory.cs`：

```csharp
using EfCore.Enterprise.Infrastructure.Data;

namespace MyApp.Infrastructure.Data;

public class MyAppDbContextFactory : DesignTimeAppDbContextFactory<AppDbContext> { }
```

### 2. 执行迁移

```bash
cd 05-Infrastructure

# 创建迁移（自动从 appsettings.json 读取连接字符串）
dotnet ef migrations add InitialCreate

# 更新数据库
dotnet ef database update

# 回到根目录启动
cd ..
dotnet run --project 06-Presentation
```

浏览器打开 `https://localhost:5001/swagger`

## 1.9 框架核心能力速览

| 你只需要做 | 框架自动帮你做 |
|-----------|--------------|
| 写实体类 | 雪花ID 生成、领域事件收集 |
| 写属性 `private set` | 通过拦截器自动填充审计字段 |
| 写 `AddDomainEvent()` | `SaveChanges` 后自动分发事件 |
| 注入 `ISuperRepository<T>` | 自动软删除过滤、租户隔离、审计追踪 |
| 配置 JWT 密钥 | 登录/鉴权/Token 刷新全自动 |
| 用 `ICacheService` | 多级缓存、BloomFilter 穿透防护 |
| 加 `[Injectable]` 标记 | 自动注册到 DI 容器 |
| 继承 `CrudAppService` | 零代码完成增删改查 |
| 继承 `BaseService` | 自动注入 `ICurrentUser`，直接获取当前用户信息 |
| 用 `Spec<T>.From(...)` | 条件自由组合，避免仓库方法爆炸 |
| 写 `BaseProfile.Configure()` | AutoMapper 自动映射 |

## 1.11 现有项目引用

如果已有项目，只需安装框架包：

```bash
cd YourExistingProject
dotnet add package EfCore.Enterprise.Application
dotnet add package EfCore.Enterprise.Infrastructure
```

然后在 `Program.cs` 中添加：

```csharp
using EfCore.Enterprise.Application.Extensions;
using EfCore.Enterprise.Infrastructure.Configuration;

var builder = WebApplication.CreateBuilder(args);

// Serilog 日志（默认带颜色）
builder.AddEfCoreSerilog();

// 一键注册（泛型版，自动注册你的 AppDbContext）
builder.Services.AddEfCoreEnterprise<YourProject.Infrastructure.Data.AppDbContext>(builder.Configuration);

// 自动注入（[Injectable] + FluentValidation + MediatR + AutoMapper）
builder.Services.AddEfCoreAutoInject(
    typeof(YourProject.Infrastructure.Data.AppDbContext).Assembly,
    typeof(Program).Assembly);

var app = builder.Build();

// 一键配置中间件管道
app.UseEfCorePipeline(app.Environment.IsDevelopment());

// 健康检查端点
app.MapHealthChecks("/health");

// 种子数据初始化
using (var scope = app.Services.CreateScope())
{
    var seeder = scope.ServiceProvider.GetRequiredService<DataSeeder>();
    await seeder.SeedAsync();
}

app.Run();
```
